# Правила генерации персонального трека

Этот документ описывает, как бот должен превращать ответы слушателя и каталог материалов в персональный маршрут обучения.

## Главный принцип

Бот не выдает полный список доступных курсов. Он собирает короткий, объяснимый и последовательный маршрут под цель слушателя.

Хороший трек отвечает на вопросы:

- с чего начать;
- почему именно с этого;
- что пройти обязательно;
- что можно пройти позже;
- какой результат будет после каждого этапа;
- какие курсы или уроки не нужны для текущей цели.

## Входные данные

### 1. Профиль слушателя

Профиль формируется из диалога с LLM. Слушатель не заполняет анкету; бот извлекает нужные поля из свободного разговора и задает только недостающие уточнения.

```json
{
  "goal": "создавать AI-агентов",
  "target_result": "собрать AI-ассистента или Telegram-бота",
  "technical_level": "beginner",
  "role": "business",
  "weekly_time": "3-5 hours",
  "depth": "balanced",
  "known_tools": ["ChatGPT"],
  "priority_topics": ["AI agents", "no-code", "Telegram bots"],
  "strategy": "quick practical result",
  "free_goal": "Хочу научиться делать AI-ассистентов для отдела продаж"
}
```

### 2. Каталог материалов

Каталог содержит курсы, модули, уроки, описания, ссылки, ДЗ и поля маршрутизации. Каталог считается изменяемым: он строится из актуального снимка Google-таблицы и имеет версию или дату синхронизации.

- уровень;
- роли;
- темы;
- технологии;
- skills;
- prerequisites;
- обязательность;
- позиция в треке;
- ориентировочное время.

При генерации трека бот должен использовать только записи со статусом `active`. Записи со статусом `deleted` не включаются в новые маршруты, а записи `new` или `changed` можно использовать только после нормализации.

## Этапы алгоритма

### Этап 0. Проверить достаточность профиля

Перед подбором материалов бот проверяет, хватает ли данных.

Минимально нужны:

- цель;
- технический уровень;
- желаемый результат;
- доступное время;
- прикладной контекст или роль.

Если не хватает одного-двух полей, бот задает короткий уточняющий вопрос. Если профиль достаточный, он строит черновой трек и может уточнять его уже после показа маршрута.

### Этап 1. Определить целевую ветку

По цели и свободному ответу бот выбирает одну или несколько веток:

| Цель слушателя | Основная ветка |
| --- | --- |
| Освоить ИИ для работы | AI-грамотность, ChatGPT, промпты |
| Делать AI-ассистентов | Промпты, AI-агенты, no-code или code |
| Автоматизировать бизнес | No-code, n8n, WeWeb, Supabase, API |
| Делать Telegram-ботов | Python, API, Telegram Bot API, backend |
| Разрабатывать AI-приложения | Python, backend, API, web, agents |
| Работать с базами знаний | RAG, embeddings, vector DB |
| Углубиться в ML/DS | Python, ML, DS, PyTorch, CV/NLP |
| Продавать AI-проекты | AI-сервисы, упаковка, продажи, проектирование |

### Этап 2. Определить стартовую точку

Старт зависит от технического уровня:

| Уровень | Старт трека |
| --- | --- |
| `zero` | базовый ChatGPT, простые промпты, обзор ИИ |
| `beginner` | no-code, Google AI Studio, промптинг, простые ассистенты |
| `beginner_to_intermediate` | AI-ассистенты, Telegram-боты, API в простом виде |
| `intermediate` | backend, API, RAG, LangGraph, интеграции |
| `advanced` | локальные LLM, сложные агентные пайплайны, production-подход |

### Этап 3. Отфильтровать материалы

Материал попадает в кандидаты, если он совпадает хотя бы по двум признакам:

- цель;
- роль;
- тема;
- технология;
- нужный результат;
- уровень;
- prerequisite для следующего важного материала.

Материал исключается или уходит в факультатив, если:

- уровень слишком высокий для текущего старта;
- тема не связана с целью;
- материал дублирует уже выбранный;
- технология не нужна для заявленного результата;
- материал требует prerequisites, которых нет в треке.

### Этап 4. Разложить материалы по этапам

Рекомендуемая структура трека:

1. Диагностика и цель.
2. База.
3. Основной навык.
4. Практическая сборка.
5. Углубление.
6. Факультативы.
7. Итоговый проект или прикладной результат.

Важно: это не контроль прогресса. LMS отвечает за прохождение и фиксацию результата. Бот только объясняет учебную последовательность.

### Этап 5. Ограничить объем

Трек должен соответствовать доступному времени:

| Время в неделю | Рекомендуемый формат |
| --- | --- |
| До 2 часов | короткий маршрут из обязательных материалов |
| 3-5 часов | базовый маршрут плюс 1-2 факультатива |
| 6-10 часов | полный маршрут с практическими заданиями |
| 10+ часов | интенсивный маршрут с углублением |

Если материалов слишком много, бот должен выбрать минимум, ведущий к цели, а остальное вынести в раздел "позже".

### Этап 6. Собрать объяснение

Каждый этап должен иметь пояснение:

- зачем этот этап нужен;
- какие материалы входят;
- какой результат ожидается;
- почему этот этап стоит перед следующим.

## Формат ответа бота

```markdown
# Ваш персональный трек

## Цель
Собрать AI-ассистента для отдела продаж без глубокого программирования.

## Логика маршрута
Сначала вы освоите базовые принципы промптов и no-code разработки, затем соберете простого AI-ассистента, после этого добавите хранение данных и автоматизацию.

## Этап 1. Быстрый старт
Материалы:
- Курс: ...
- Урок: ...

Результат:
Вы поймете базовую логику AI-ассистента и сможете собрать первый прототип.

## Этап 2. Основной навык
Материалы:
- Курс: ...
- Урок: ...

Результат:
Вы сможете спроектировать диалог, собрать данные пользователя и связать интерфейс с AI-логикой.

## Обязательные материалы
- ...

## Дополнительные материалы
- ...

## Что можно пропустить сейчас
- ...

## Ориентировочная длительность
При нагрузке 3-5 часов в неделю: 6-8 недель.

## Актуальность
Маршрут построен по каталогу курсов от 2026-06-04.
```

## Правила хорошего трека

- В начале не должно быть слишком сложных материалов.
- Каждый следующий этап должен опираться на предыдущий.
- Факультативы не должны раздувать основной маршрут.
- Технические материалы должны появляться только тогда, когда без них нельзя достичь цели.
- Бот должен объяснять исключения: почему часть курсов не включена в текущий маршрут.
- Для разных ролей один и тот же курс может иметь разный статус: обязательный, дополнительный или пропускаемый.
- Бот не должен имитировать опрос: уточняющие вопросы задаются по одному и только по делу.
- Если каталог обновился, новый трек должен строиться по новой версии каталога.
- Если ранее выданный трек содержит удаленный материал, при следующем обращении бот должен предложить замену.

## Пример маршрутизации

### Сценарий: начинающий хочет делать AI-ассистентов для продаж

Профиль:

```json
{
  "goal": "AI-ассистенты",
  "role": "sales",
  "technical_level": "beginner",
  "weekly_time": "3-5 hours",
  "depth": "balanced"
}
```

Маршрут:

1. База ChatGPT и промптов.
2. Google AI Studio или no-code старт.
3. Диалоговый AI-ассистент.
4. Сбор данных из диалога.
5. AI-консультант или нейро-продажник.
6. Факультатив: n8n/WeWeb, если нужна автоматизация.
7. Факультатив: Telegram-бот, если нужен мессенджер.

### Сценарий: разработчик хочет делать AI-агентов

Профиль:

```json
{
  "goal": "AI-агенты",
  "role": "developer",
  "technical_level": "intermediate",
  "weekly_time": "6-10 hours",
  "depth": "deep"
}
```

Маршрут:

1. Краткий обзор промпт-архитектуры.
2. API и backend для AI-приложений.
3. Telegram-бот или веб-приложение как интерфейс.
4. Память и хранение данных.
5. RAG и базы знаний.
6. LangGraph или агентные пайплайны.
7. Локальные модели и production-интеграции как углубление.
