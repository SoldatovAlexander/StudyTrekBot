# Telegram/FastAPI MVP

## Роль Telegram-бота

Telegram-бот — это интерфейс диалога со слушателем. Он не заменяет LMS и не ведет обучение. Его задача:

- принять цель слушателя в свободной форме;
- задать несколько уточняющих вопросов;
- сформировать профиль слушателя;
- запросить генерацию персонального трека;
- показать маршрут понятным текстом;
- дать возможность уточнить или пересобрать трек.

## Почему Telegram подходит для MVP

- не нужен отдельный frontend;
- формат естественно поддерживает диалог с LLM;
- можно быстро протестировать гипотезу на реальных слушателях;
- легко отправлять длинные структурированные ответы;
- можно добавить кнопки для простых действий: "уточнить", "сократить", "углубить", "показать факультативы".

## Backend

Backend: Python/FastAPI.

Telegram framework: aiogram 3.x.

FastAPI отвечает за:

- прием webhook от Telegram;
- хранение состояния диалога;
- работу с OpenRouter;
- доступ к нормализованному каталогу;
- генерацию персонального трека;
- синхронизацию Google Sheets;
- служебные API для администрирования.

## Схема MVP

```text
Telegram
  ↓ webhook
FastAPI + aiogram 3.x
  ↓
Dialog Manager
  ↓
OpenRouter / DeepSeek V4 Flash free
  ↓
Track Builder
  ↓
Normalized Catalog
  ↑
Google Sheets Sync
```

## Основные компоненты

### 1. Telegram Adapter

Принимает сообщения и отправляет ответы.

Реализация: aiogram 3.x.

Задачи:

- обработать входящий webhook;
- определить пользователя;
- передать текст в диалоговый менеджер;
- отправить ответ пользователю;
- поддерживать простые команды.

Минимальные команды:

```text
/start
/newtrack
/help
```

### 2. Dialog Manager

Ведет разговор и собирает профиль слушателя.

Задачи:

- хранить текущий профиль;
- определять, каких данных не хватает;
- задавать уточняющие вопросы;
- решать, когда можно строить трек;
- передавать профиль в Track Builder.

### 3. LLM Client

Клиент OpenRouter.

Параметры:

```text
OPENROUTER_API_KEY=...
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=deepseek/deepseek-v4-flash:free
```

Модель должна задаваться через конфигурацию, потому что бесплатные модели и лимиты могут меняться.

### 4. Catalog Sync

Синхронизирует Google Sheets с локальным каталогом.

Задачи:

- читать список вкладок;
- читать строки с занятиями;
- считать `source_hash`;
- определять `new`, `changed`, `deleted`;
- сохранять версию синхронизации;
- передавать новые и измененные записи на нормализацию.

### 5. Normalized Catalog

Хранит подготовленные данные для маршрутизации.

Для MVP используем SQLite. PostgreSQL оставляем для развития проекта.

Минимальные таблицы:

```text
catalog_versions
raw_lessons
normalized_lessons
student_profiles
generated_tracks
```

### 6. Track Builder

Собирает персональный трек.

Задачи:

- подобрать материалы под профиль;
- отсортировать по уровню и prerequisites;
- ограничить объем под доступное время;
- выделить обязательное и факультативное;
- сформировать объяснение маршрута.

## Минимальные эндпоинты FastAPI

```text
POST /telegram/webhook
POST /admin/sync-catalog
GET  /admin/catalog/status
POST /tracks/generate
GET  /tracks/{track_id}
```

Для MVP публичным может быть только `/telegram/webhook`. Остальные эндпоинты можно оставить служебными.

## Сценарий диалога

```text
Слушатель: Хочу научиться делать AI-ассистентов для продаж.

Бот: Хорошая цель. Уточню пару вещей, чтобы не давать лишние курсы. Вам нужен путь без программирования или готовы немного изучать Python/API?

Слушатель: Лучше без программирования, но если нужно, могу немного разобраться.

Бот: Понял. Сколько времени в неделю реально готовы выделять?

Слушатель: 3-4 часа.

Бот: Тогда соберу практический маршрут: промпты, no-code прототип, диалоговый ассистент, сбор данных, а Telegram-бот и RAG вынесу как развитие.
```

## Формат ответа в Telegram

Ответ должен быть короче, чем полноценный документ, но достаточно структурированным:

```text
Ваш персональный трек

Цель: AI-ассистент для продаж без глубокого программирования.

Логика: сначала быстрый прототип и промпты, затем диалоговый сценарий, потом сбор данных и автоматизация.

Этап 1. База
...

Этап 2. Первый ассистент
...

Обязательное:
...

Факультативно:
...
```

## Границы MVP

В MVP не делаем:

- личный кабинет;
- контроль прохождения;
- оценку домашних заданий;
- сложную аналитику;
- оплату;
- полноценную LMS-интеграцию;
- автоматическую перестройку трека без запроса слушателя.
- Telethon как основной Telegram-инструмент.

## Развитие

После MVP Telegram-бот может перейти от подбора готовых курсов к сборке персонального курса из отдельных занятий.

Это значит, что Track Builder будет работать не только на уровне курсов, но и на уровне уроков:

- выбирать занятия из разных курсов;
- соединять их в новую последовательность;
- строить ветки развития цели;
- обновлять маршрут при изменении целей слушателя;
- исключать дубли;
- формировать индивидуальную программу обучения.
