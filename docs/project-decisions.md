# Фиксация основных решений проекта

Дата фиксации: 2026-06-04.

## 1. Проблема

В Университете Искусственного Интеллекта есть много курсов и учебных материалов по ИИ. При покупке отдельного курса программа понятна. При покупке пакета курсов слушатель получает большой набор материалов, но не получает персональный маршрут. Из-за этого он может путаться, переключаться между курсами и терять фокус.

Ключевая проблема: не недостаток материалов, а отсутствие персональной навигации по ним.

## 2. Цель проекта

Создать бота-навигатора, который через естественный диалог с LLM выясняет цель, уровень и контекст слушателя, а затем формирует понятный персональный трек обучения по доступным курсам и материалам.

## 3. Границы проекта

Бот отвечает за персональный маршрут:

- что проходить;
- в каком порядке;
- почему выбран такой порядок;
- какие материалы обязательны;
- какие материалы факультативны;
- какой результат ожидается после каждого этапа.

Бот не заменяет LMS.

Вне рамок MVP:

- контроль прохождения;
- проверка домашних заданий;
- выставление оценок;
- ведение журнала прогресса;
- личный кабинет;
- платежи;
- полноценная LMS-интеграция.

## 4. Источник данных

Основной источник данных — Google Sheets с курсами и материалами.

Таблица может меняться:

- курсы могут добавляться;
- занятия могут добавляться;
- описания и ссылки могут меняться;
- материалы могут удаляться.

Поэтому проект должен использовать динамический каталог:

- регулярная синхронизация Google Sheets;
- сохранение сырого слоя данных;
- нормализованный слой для маршрутизации;
- статусы записей `active`, `new`, `changed`, `deleted`, `needs_review`;
- версия каталога, по которой построен трек.

## 5. Диалог вместо анкеты

Диагностика слушателя должна проходить в формате естественного разговора, а не в виде жесткого опроса.

LLM внутри диалога заполняет профиль:

- цель;
- желаемый результат;
- технический уровень;
- роль или профессиональный контекст;
- доступное время;
- желаемую глубину;
- знакомые инструменты;
- приоритетные направления;
- ограничения.

Бот задает только необходимые уточнения и не должен превращать разговор в длинную форму.

## 6. MVP-интерфейс

Для MVP выбран Telegram-бот.

Telegram подходит, потому что:

- не требует отдельного frontend;
- естественно поддерживает диалог;
- быстро запускается на пилотной группе;
- позволяет отправлять структурированные маршруты;
- хорошо подходит для проверки гипотезы.

## 7. Технический стек MVP

Принятый стек:

- язык: Python;
- backend: FastAPI;
- Telegram framework: aiogram 3.x;
- LLM provider: OpenRouter;
- LLM model: `deepseek/deepseek-v4-flash:free`;
- источник каталога: Google Sheets;
- хранилище MVP: SQLite.

PostgreSQL оставлен для развития и масштабирования.

Telethon не используется как основной инструмент MVP. Его можно рассмотреть позже только для задач чтения каналов, истории чатов или автоматизации от имени пользовательского аккаунта.

## 8. Архитектура MVP

```text
Telegram user
  ↓
aiogram 3.x
  ↓
FastAPI webhook
  ↓
Dialog Manager
  ↓
OpenRouter / DeepSeek V4 Flash free
  ↓
Track Builder
  ↓
Normalized Catalog
  ↑
Google Sheets Sync
```

## 9. Минимальные компоненты

MVP должен включать:

- Telegram Adapter;
- Dialog Manager;
- OpenRouter client;
- Google Sheets Sync;
- Raw Catalog;
- Normalized Catalog;
- Track Builder;
- SQLite-хранилище;
- сохранение профиля слушателя и сгенерированного трека.

## 10. Формат результата для слушателя

Бот должен выдавать не список всех курсов, а маршрут:

- цель трека;
- логика выбора;
- этапы;
- курсы, модули или уроки;
- обязательные материалы;
- факультативные материалы;
- материалы, которые можно пропустить сейчас;
- ожидаемый результат;
- ориентировочная длительность;
- версия или дата актуальности каталога.

## 11. Развитие проекта

После MVP проект должен развиваться от подбора готовых курсов к сборке персонального курса из отдельных занятий.

Это означает, что система сможет:

- брать занятия из разных курсов;
- выстраивать их в новую последовательность;
- учитывать prerequisites;
- исключать дубли;
- добавлять ветки развития цели;
- пересобирать маршрут при изменении цели слушателя;
- формировать индивидуальную программу обучения.

Пример развития цели:

```text
AI-ассистент для продаж
  → диалоговый ассистент
  → сбор данных
  → Telegram-бот
  → память
  → RAG
  → интеграции
```

## 12. Следующий практический шаг

Следующий шаг — перейти от проектной фиксации к прототипированию:

1. создать структуру Python/FastAPI проекта;
2. добавить aiogram webhook;
3. добавить SQLite-схему;
4. сделать чтение тестового каталога;
5. реализовать короткий диалог;
6. сгенерировать первый персональный трек на тестовых данных.
